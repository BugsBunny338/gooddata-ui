Table Totals in ExecutionObject

There are several typical examples how to define table totals, how they looks like on the UI and what has to be defined in ExecutionObject (AFM and ResultSpec) to achieve expected results.

Currently Supported Types

TypeIn 1st dimensionIn 2nd dimension
Grand totalsDONENOT YET
Sub-totalsNOT YETNOT YET

Grand Totals

Grand totals are totals computed for a measure in a whole column or row.

In 1st dimension

  • a1, a2 are attributes.
  • m1 is a measure.

Example: ExecutionObject for Grand Totals in 1st dimension

execution : {
    afm: {
        ...
        nativeTotals: [
            {
                measureIdentifier: 'm1',
                attributeIdentifiers: []
            }
        ]
    },
    resultSpec: {
        ...
        dimensions: [
            {
                itemIdentifiers: ['a1', 'a2'],
                totals: [
                    {
                        measureIdentifier: 'm1',
                        type: 'sum',
                        attributeIdentifier: 'a1'
                    },
                    {
                        measureIdentifier: 'm1',
                        type: 'nat',
                        attributeIdentifier: 'a1'
                    }
                ]
            },
            {
                itemIdentifiers: ['measureGroup'],
                totals: []
            }
        ]
    }
}
